[ PHASE 1: RECONNAISSANCE ]

THEORY: SQL INJECTION

SQL Injection occurs when untrusted data is concatenated into database queries. Attackers can manipulate the query structure to bypass checks or retrieve data.

OBJECTIVE

Login as admin without knowing the password.

HINT 1: QUERY STRUCTURE

The backend likely processes login like this:

SELECT * FROM users WHERE username = '$user' AND password = '$pass'

Your goal is to make this query verify as TRUE even if you don't know the password.

HINT 2: BOOLEAN LOGIC

If you set the username to something that creates a tautology (always true statement), you can bypass the password check.

Try injecting a quote ' to close the string, then add OR '1'='1.

FULL EXECUTION PLAN

Enter the following as the username:

admin' OR '1'='1

This changes the query to:

SELECT * ... WHERE username = 'admin' OR '1'='1' ...

Since '1'='1' is true, the database returns the admin record regardless of the password.
